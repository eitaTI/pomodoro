# LLM INDEX - Template Context

## Directory Purpose
This folder is the "boilerplate" for the project. Users will duplicate this folder to their own named workspace. 

## Structure
- `backend/`: A scaffolded NestJS API project. Contains a `docker-compose.yml` to spin up a local PostgreSQL instance.
- `frontend/`: A scaffolded Next.js (App Router) project with Tailwind CSS configured.

## Agent Behavior
- Do NOT make direct edits to this folder when helping a student complete a lesson. You must instruct the student to edit files inside their OWN duplicated folder (e.g. `/lucas/backend/` instead of `/_template/backend/`).
- If you are asked to add a new library or structural feature to the baseline project for *future* students, then you should modify this `_template` folder.
